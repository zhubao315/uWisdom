# 开源选型

所属领域：[开源发烧友]({{ '/knowledge-base/domains/open-source-enthusiast.html' | relative_url }})

## 定义

评估开源项目的成熟度、社区活跃度、维护质量和长期可持续性，做出技术选型决策。

## 评估框架

### 成熟度指标

- GitHub Stars / Forks / Contributors 数量趋势
- Release 频率与版本稳定性
- Issue 响应时间与关闭率
- 文档完整度与更新频率

### 风险评估

- 许可证类型（MIT / Apache / GPL / 商业限制）
- 主要维护者是否为单一公司
- 是否有明确的商业支持或赞助
- 依赖链深度与安全漏洞历史

### 决策矩阵

- 功能覆盖度 vs 学习成本
- 社区规模 vs 代码质量
- 短期可用性 vs 长期演进能力
