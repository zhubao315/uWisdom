# 开源协作

所属领域：[开源发烧友]({{ '/knowledge-base/domains/open-source-enthusiast.html' | relative_url }})

## 定义

参与开源社区的协作模式、贡献流程和最佳实践，以及如何将开源文化引入团队。

## 核心要点

### 贡献方式

- Issue 报告：复现步骤、环境信息、期望行为
- 代码贡献：Fork → Branch → PR → Review → Merge
- 文档改进：翻译、补充示例、修正错误
- 社区支持：回答问题、撰写教程

### 社区治理

- CODEOWNERS 机制与维护者职责
- RFC（Request for Comments）流程
- 行为准则（Code of Conduct）

### 团队引入

- 内部开源文化：代码透明、文档公开、异步协作
- 20% 时间贡献外部项目
- 内部技术雷达跟踪开源趋势
