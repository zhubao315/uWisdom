# 云转型

所属领域：[云计算与云原生]({{ '/knowledge-base/domains/cloud-native.html' | relative_url }})

## 定义

企业从传统 IT 基础设施和单体架构向云原生架构转型的路径规划、技术选型和组织变革方法。

## 核心要点

### 转型路径

- **评估阶段**：现状盘点、应用分类、依赖梳理
- **试点阶段**：选择低风险应用先行迁移
- **规模化**：建立平台能力，加速迁移节奏
- **优化阶段**：持续优化成本、性能和可靠性

### 迁移策略

- Rehost（搬站）：直接迁移到云上，最小改动
- Replatform（适配）：小幅改造以利用云服务
- Refactor（重构）：容器化、微服务化改造
- Rebuild（重建）：基于云原生重新设计

### 组织变革

- DevOps 文化：开发与运维融合
- 平台工程：建设内部开发者平台（IDP）
- 技能转型：团队培训与新角色引入

### 风险管理

- 数据迁移的完整性与一致性
- 业务连续性保障
- 成本控制：避免云上费用失控
