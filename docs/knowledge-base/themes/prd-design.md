# PRD 设计

所属领域：[产品经理]({{ '/knowledge-base/domains/product-manager.html' | relative_url }})

## 定义

产品需求文档（PRD）的结构化设计方法，将用户需求、业务目标和技术约束转化为清晰、可执行的需求描述。

## 核心要点

### PRD 结构

- 背景与目标：为什么做、要达成什么
- 用户故事：谁、在什么场景、需要什么
- 功能需求：具体功能点、交互流程、边界条件
- 非功能需求：性能、安全、兼容性
- 验收标准：可测试、可量化的完成定义

### 设计原则

- 需求与方案分离：PRD 描述"做什么"，不规定"怎么做"
- 优先级明确：P0/P1/P2 分级，控制范围
- 可追溯：需求来源、决策依据、变更记录

### 常见问题

- 需求膨胀：不断追加功能导致交付延迟
- 验收模糊：缺乏可量化标准导致反复确认
- 文档过时：需求变了文档没更新
