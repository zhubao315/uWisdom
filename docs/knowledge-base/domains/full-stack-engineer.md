# 全栈工程师

## 领域定义

具备前端、后端、数据库、API 设计与部署运维全链路能力，能够独立交付完整功能模块或小型产品。保持对真实系统实现的控制力，让架构判断不脱离工程事实。

## 对你的价值

- 全栈能力帮助你理解系统全貌，而非只看局部
- 是架构师的重要基础：只有理解每一层，才能做出合理取舍
- 在 AI 时代，全栈能力扩展到"模型 + 数据 + 应用 + 部署"的完整链路

## 核心关注方向

### 前端工程

- 现代前端框架（React / Vue / Svelte）
- 组件化设计与状态管理
- 性能优化与用户体验

### 后端工程

- API 设计（REST / GraphQL / gRPC）
- 微服务架构与服务拆分
- 数据库设计与查询优化

### 全链路交付

- 从需求到上线的完整交付流程
- 自动化测试与持续集成
- 监控告警与线上问题排查

### AI 时代的全栈

- AI 应用的前后端架构
- 模型服务化与 API 网关
- 用户交互与 AI 能力的结合设计

## 典型 Project

- 个人知识系统工程化（uWisdom）
- Agent 工具链搭建
- 全链路原型验证
- 快速原型开发与内部工具建设
