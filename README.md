> **核心理念**：成果源于**挖掘机会**，而非解决问题。—— 德鲁克《成果管理》（1964）

---

## 📌 系统定位

**uWisdom 百科**是 Heartbeat-Like-A-Man 系统的一部分，是一个**持续迭代、深入研究与更新**的领域知识百科。

**目标**：
1. 设定关注的领域，不断探寻「领域」路径
2. 保持正确迭代并跟随时代步伐
3. 成为有趣的人
4. 持续反思、探索新知、完善这个领域知识百科

---

## 🧭 四大核心身份

| 身份 | 定位 | 核心能力 |
|------|------|----------|
| **∑ 架构师** | 设计和规划复杂系统 | 系统思维、架构设计、技术整合 |
| **∑ 投资人** | 分析市场、管理风险 | 机会挖掘、风险配置、财富增值 |
| **∑ 终身学习者** | 持续追求知识与技能 | 跨学科学习、知识内化、技能迭代 |
| **∑ 生活艺术家** | 发现和创造美好 | 审美力、创造力、生活品质 |

## 🦞 与 Heartbeat-Like-A-Man 的整合

### 系统协同

| Heartbeat 组件 | uWisdom 联动 |
|----------------|------------|
| **微触发管理器** | 检查 uWisdom 是否有新洞察需更新 |
| **梦境思考** | 深度探索 uWisdom 领域的连接点 |
| **自主探索** | 主动挖掘 uWisdom 领域的机会 |
| **思考记录** | 存入 uWisdom/08-思考记录/ |

### 持续迭代机制

```
1. 用户分享 → 2. 系统记录 → 3. 梦境思考 → 4. 提炼洞察 → 5. 更新百科 → 6. 等待验证 → 回到 1
```

## 🤖 系统使用指南

### 如何更新百科

1. **新洞察** → 存入 `08-思考记录/日期 - 主题.md`
2. **领域知识** → 存入对应 `0X-领域/子领域/`
3. **项目进展** → 更新对应项目文档
4. **系统变更** → 记录到 `09-系统日志/`

### 如何反思

1. 每日/每周/每月检查 `08-思考记录/`
2. 梦境思考产生洞察后自动记录
3. 定期回顾 `07-职业历程/` 进行对标
